# ⚡ Intel Dashboard — Intelligence Report Viewer

A web dashboard for viewing and browsing intelligence reports generated by the LeightonSec Intel Pipeline. Analysts can read reports in the browser, filter by version and time period, search across reports, and monitor which sources are actively feeding intelligence.

---

## What It Does

The Intel Pipeline generates markdown reports twice daily and saves them locally and to Obsidian. This dashboard surfaces those reports in a clean browser interface — giving analysts a single place to read, search and browse all gathered intelligence without opening Obsidian or navigating the filesystem.

---

## Features

- **Report viewer** — read full intel reports rendered from markdown in the browser
- **Version filtering** — filter between v1 and v2 pipeline reports
- **Period filtering** — filter morning and evening runs separately
- **Search** — search reports by date or keyword
- **Source status** — see which feed categories are active and how many sources per category
- **Stats overview** — total reports, v1/v2 breakdown, total sources
- **Auto-refresh** — stats and report list update every 60 seconds

---

## Source Categories

Reports are organised by the four intel categories from the pipeline:

| Category | Colour | Sources |
|----------|--------|---------|
| Security | Red | Krebs, Hacker News, BleepingComputer, Dark Reading, SecurityWeek |
| AI Research | Blue | arXiv cs.AI, arXiv cs.CR |
| BSV & Bastion | Amber | CoinGeek, BitcoinSV.io |
| CVE | Orange | CISA Alerts, CISA Activity, Exploit-DB |

## Setup

**Requirements:** Python 3.x, intel-pipeline running and generating reports

```bash
# Clone the repo
git clone git@github.com:LeightonSec/intel-dashboard.git
cd intel-dashboard

# Create virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Set secret key
echo "SECRET_KEY=your-secret-key-here" > .env

# Run the dashboard
python3 app.py
```

Then open `http://127.0.0.1:5004` in your browser.

**Note:** The dashboard reads reports from `~/Documents/Projects/intel-pipeline/reports/` and `~/Documents/MyVault/Inbox/`. Ensure the intel pipeline has generated at least one report before opening the dashboard.

## API Endpoints

- `GET /api/reports` — list all reports, supports ?version= and ?category= filters
- `GET /api/reports/<filename>` — get full report content as HTML
- `GET /api/sources` — get configured sources from pipeline config
- `GET /api/stats` — dashboard statistics
